Project Goal & Purpose

Create a professional portfolio and booking web application for Madison's Photography that allows prospective customers to review services, view a gallery of sample work, and request bookings via a client-side booking form. This overview explains the requirements, content categories, user personas, and the interactive features planned for the site.

Users & Personas

  1. Clients: Individuals or families seeking portrait or event photography (e.g., family sessions, senior portraits).
  2. Couples: Brides, grooms, and their families planning wedding photography.
  3. Local organizations: Community groups or small businesses that need event or promotional photography.
  4. Friends/Family: Support and referrals; may share the portfolio on social media.

Content Categories

  • About Madison — experience, biography, and approach.
  • Services — packages, pricing, and sample timelines.
  • Portfolio/Gallery — curated images for portraits, weddings, events.
  • Booking & Contact — forms to request a quote or a booking.
  • Testimonials & Work Experience — samples from past jobs and client reviews.

Planned Interactivity & Dynamic Functionality

My site uses JavaScript, jQuery, and jQuery UI to create a dynamic and interactive user experience without requiring a server-side backend (read-only operations only):

  • AJAX-loaded gallery — images and metadata are read from a JSON file (`data/gallery.json`). This showcases Madison's work and allows gallery updates without code edits to the HTML.
  • jQuery UI Accordion — used on the 'Services' descriptions to collapse/expand service details and pricing.
  • jQuery UI Datepicker — included in the booking form to select event dates quickly and accurately.
  • jQuery UI Dialogs — used to display full-size images in a modal and to show booking confirmations or validation errors.
  • Form validation — a client-side booking form validates required fields; submission is simulated and displayed in a confirmation dialog (no server-side CRUD).

Where these appear:

  • Gallery: This page (`project_overview.html`) includes the gallery (interactive thumbnails, modal viewer).
  • Booking & Contact: The contact and booking forms are accessible on `contact_me.html` and use date picker and client-side validation — Go to Contact.
  • Services: The Services page uses an accordion for the list of packages and details.

Examples of inspiration:

  • Photo galleries and modals like those on typical portfolio websites such as Squarespace or Adobe Portfolio (visual inspiration).
  • Booking forms with date pickers like event photographer websites and appointment booking widgets.

Accessibility & Performance Notes

The site uses semantic HTML, accessible form labels, and clear alternative text for images loaded from the JSON gallery. The gallery uses optimized thumbnails for faster page loads and only loads full images when requested by the user via the modal dialog.

  • jQuery UI widgets (accordion, datepicker, dialogs)
